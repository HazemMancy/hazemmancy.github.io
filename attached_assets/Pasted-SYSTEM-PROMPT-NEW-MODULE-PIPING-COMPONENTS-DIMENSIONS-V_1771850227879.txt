CORE UI/UX REQUIREMENTS
- “Dataset Status” always visible: Loaded / Not loaded, dataset name, last updated, source note.
- If dataset not loaded: show importer + templates + clear message “Upload licensed ASME/manufacturer data.”
- Filters must feel like engineering software:
  - NPS (inch) / DN (mm) toggle
  - Pressure rating/Class (150/300/600/900/1500/2500) where applicable
  - Schedule/Wall (Sch 10/40/80/160, XS/XXS) where applicable
  - Type selection (e.g., WN/SO/BL flanges; LR/SR elbows; BW/SW/THD; etc.)
- Outputs:
  - Dimensions table (from dataset)
  - Derived geometric values (area, volume, mass estimate if density provided)
  - 2D dimensioned sketch (SVG) with labels matching returned dimensions
  - Export: JSON of selected record + SVG/PNG + PDF report

CATEGORY REQUIREMENTS (WHAT EACH PAGE MUST DO)

1) FLANGES
- Support flange types: WN, SO, BL (at minimum)
- Inputs: NPS/DN, Class, facing, bore (if dataset includes)
- Output: key dims (OD, bolt circle, bolt holes, thickness, hub dims, RF height if provided)
- Visual: flange cross-section sketch with labeled dims

2) FITTINGS
- Types: 90 LR elbow, 45 elbow, tee, reducer (concentric/eccentric)
- Inputs: NPS1/NPS2, schedule, end type (BW/SW/THD)
- Output: center-to-end, center-to-center, overall lengths; wall where provided
- Visual: elbow/tee/reducer sketch with labeled dims

3) GASKETS
- Types: spiral wound, ring type, full-face (as dataset allows)
- Inputs: NPS, Class, gasket type
- Output: ID/OD, thickness
- Visual: gasket ring sketch

4) VALVES
- Types: gate, globe, ball, check (catalog-level)
- Inputs: NPS, Class, valve type, end connection
- Output: face-to-face, end-to-end, height (if dataset includes)
- Visual: simplified valve envelope sketch + key dims

5) LINE BLANKS
- Types: spectacle blind, spade, spacer (if dataset)
- Inputs: NPS, Class
- Output: thickness, OD/ID, handle dims (if provided)
- Visual: spectacle blind sketch

6) PIPE
- Inputs: NPS/DN, schedule
- Output: OD, ID, wall thickness, weight per length (if dataset provides density or weight)
- Visual: pipe section sketch with OD/ID/t labels

7) OLETS
- Types: weldolet, sockolet, threadolet
- Inputs: run size, branch size, rating, type
- Output: key dims as provided
- Visual: olet branch connection sketch

8) PIPE FLEXIBILITY
- Provide a calculation tool for expansion loop screening and thermal growth.
- Inputs: material (E, alpha), temperature change, run length, restraint assumption, allowable stress (user provided)
- Outputs: thermal expansion, basic stress screening
- Must be clearly labeled: “Screening tool; detailed flexibility/stress analysis requires Caesar II/Autopipe.”
- References panel: ASME B31.3 concepts only (no proprietary tables), and disclaimers.

9) SAFE SPANS
- Provide a safe span screening tool that REQUIRES either:
  - user-provided safe span table dataset, OR
  - user-provided allowable deflection + pipe properties.
- Outputs: recommended span + deflection check (screening).
- Must label: “Screening; final support spacing per project piping spec/stress analysis.”
